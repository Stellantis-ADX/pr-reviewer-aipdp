# Fix Summary: Comments Not Being Posted to GitHub

## Problem

Code review comments generated by Claude were not being posted to GitHub PRs. The log showed:
- `Found comment separator` (comments were generated)
- `No reviews found` (comments weren't being parsed)
- `Submitting empty review is disabled` (no comments posted)

## Root Cause

**Regex Pattern Mismatch**

The prompt example showed line number format: `22 - 22:` (WITH spaces)
But the parser regex expected format: `22-22:` (NO spaces)

Claude followed the prompt example and generated `63 - 65:`, but the parser couldn't recognize it.

### Files Affected

1. **[core/schemas/review.py:129](core/schemas/review.py#L129)** - Main regex pattern
2. **[core/schemas/review.py:164](core/schemas/review.py#L164)** - Extraction regex pattern

### Prompt Example (core/templates/prompts.py:145)
```
22 - 22:
There's a syntax error in the add function.
```

### Old Parser Regex (BROKEN)
```python
line_number_range_regex = r"(?:^|\s)(\d+)-(\d+):\s*$"
```

This required NO spaces: `63-65:`

### New Parser Regex (FIXED)
```python
line_number_range_regex = r"(?:^|\s)(\d+)\s*-\s*(\d+):\s*$"
```

This allows optional spaces: `63 - 65:` or `63-65:`

## Fix Applied
